Vue 路由（Vue Router）

Vue 提供了官方的路由库 Vue Router，用于构建单页面应用（SPA，Single Page Application）

Vue Router 可以让应用在不刷新的情况下根据不同的 URL 显示不同的页面

Vue Router 核心功能：

  • 在单页面应用中实现页面导航
  • 根据 URL 动态加载对应的组件
  • 支持浏览器前进、后退等功能
  • 提供路由守卫用于权限控制和数据预加载

Vue 状态管理（Vue State Management）

Vue 状态管理（State Management）是指在 Vue 应用中集中式地管理组件之间共享的数据状态。随着应用变得复杂，仅依赖组件之间的 props 和 events 通信已经不足以满足需求，这时候就需要引入一种更清晰、可维护的状态管理方案

Vue 常见的状态管理方案包括：

  1. Vuex（适用于 Vue 2 & Vue 3）
  2. Pinia（Vue 3 官方推荐）
  3. Vue 组合式 API + reactive 共享状态

Pinia

Pinia 是 Vue 官方推荐的新一代状态管理库，它的设计目标是 更轻量、更直观、更易用

Pinia 其名字来源 “菠萝”（西班牙语 pina）

为什么要用 Pinia

在 Vue 中，简单的项目可以用 props 和 emit 进行组件通信，但当项目规模扩大后，就会遇到以下痛点：

  • 跨层级通信麻烦（props drilling）
  • 兄弟组件通信不方便
  • 状态分散（不同组件各自维护，难以管理）

Pinia 作为集中式状态管理，可以帮我们解决这些问题

Pinia 的特点

  • 更轻量：API 简单易学
  • 更直观：和 Vue3 的 Composition API 风格一致
  • 完整的 TypeScript 支持：类型推导更有好
  • 插件系统：支持持久化存储（如 localStroage）、中间件扩展等

Pinia 常用 API

$reset()

将 store 恢复到初始状态（定义时的初始值），但使用组合式 setup 函数定义的 Store 不能使用该方法（需要手动实现 $reset 方法）

$patch()

一次性批量修改多个 state 属性
避免多次触发响应式更新

$subscribe()

监听整个 state 的变化

$onAction()

监听 action 的调用

storeToRefs()

将 store 的响应式 state 和 getter 转为 refs，防止解构丢失响应式
